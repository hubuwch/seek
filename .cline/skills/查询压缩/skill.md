# 查询压缩技能

## 目的
将用户的冗长问题转换为简洁的关键词查询，让搜索引擎能够精确匹配，减少噪音，提升首页结果的相关性。

## 使用时机
- 调用任何搜索 API 之前
- 用户问题包含口语化填充词时
- token 成本需要控制时

## 指令

### 第一步：识别答案类型
检测用户需要的答案类型：

- `定义` - 关键词：什么是、定义、含义、意思
- `步骤` - 关键词：如何、怎么、步骤、方法、实现
- `数据` - 关键词：多少、数据、销量、统计、占比
- `对比` - 关键词：区别、对比、vs、差异
- `故障排查` - 关键词：错误、失败、问题、无法、报错
- `列举` - 关键词：哪些、列举、包括、分为、分类

### 第二步：提取核心关键词
- 移除疑问词（如何、为什么、什么是、请问）
- 移除标点和停用词（的、了、吗、呢）
- 保留名词、动词、数字、专有名词
- 最多保留 5-7 个关键词

### 第三步：格式化输出
输出一个 JSON 对象，包含：
- `compressed_query`：空格分隔的关键词
- `answer_type`：上述类型之一
- `token_saved_estimate`：预估节省百分比

## 输入格式
用户问题：<自然语言问题>

## 输出格式
```json
{
  "compressed_query": "关键词1 关键词2 关键词3",
  "answer_type": "定义|步骤|数据|对比|故障排查|列举",
  "token_saved_estimate": "XX%"
}
```
## 边界情况
- 如果问题无法识别，返回错误
- 如果问题本身已经很短（< 5 个词），保持不变
- 如果问题包含代码或技术术语，原样保留
- 如果答案类型不明确，默认使用 定义
